# Static timer shell

## Parent PRD

#5

## What to build

Create the first end-to-end version of the timer app shell as a single-page browser experience. This slice should deliver the default view described in the parent PRD: a readable personal timer interface that opens in Work mode at `25:00`, shows the active mode label, countdown, primary action area, reset control area, and status-message area, and remains usable on small screens. This issue covers the base user-facing structure and presentation only, not active countdown behavior.

## Acceptance criteria

- [ ] The app renders as a single-page experience with a visible mode label, countdown display, primary action button, reset button, and status-message area.
- [ ] On first load, the UI starts in Work mode and shows `25:00`.
- [ ] The layout is readable and operable on desktop and mobile-sized browser widths.
